Create an animated team showcase component that displays a 2x2 grid of team member cards with an intelligent scanning border effect that highlights each member sequentially.

Animation System
Timing Constants

SCAN_DURATION: 2 seconds per card
TOTAL_DURATION: 8 seconds (2s Ã— 4 cards)
FRAME_SEQUENCE: [0, 0, 1, 1, 2, 2, 3, 3, 0] - defines which card index the border should be positioned on
FRAME_KEYFRAME_TIMES: [0, 0.07, 0.25, 0.32, 0.5, 0.57, 0.75, 0.88, 1] - normalized time points (0-1) for border animation
CARD_SEQUENCE: [0, 0, 1, 1, 2, 2, 3, 3, 0] - defines which card index should be active (colored/visible tag)
CARD_KEYFRAME_TIMES: Same as FRAME_KEYFRAME_TIMES for synchronization
SMOOTH_EASE: [0.45, 0, 0.55, 1] - custom easing curve for smooth animations
MEASUREMENT_DEBOUNCE_MS: 100ms debounce delay for DOM measurements

Animation Logic
Image Animations (per card)
For each of 4 cards, create animation based on its index:

Filter (grayscale):

Map through CARD_SEQUENCE
If position matches card index: grayscale(0%) (full color)
Otherwise: grayscale(100%) (grayscale)


Scale:

Active card: 1.05
Inactive cards: 1.0


Transition: duration = TOTAL_DURATION, times = CARD_KEYFRAME_TIMES, ease = SMOOTH_EASE, infinite loop

Bio Tag Animations (per card)

Opacity:

Active card: 1 (visible)
Inactive cards: 0 (hidden)


Y-translation:

Active card: 0 (normal position)
Inactive cards: 12 (slightly down)


Same transition config as images

Scanning Border Animation
Complex feature requiring DOM measurements with debouncing:

Setup Phase (useEffect + useRef):

Create refs for: grid container, all 4 card elements, measurement timeout
Debounced measurement function to prevent excessive updates during animation
Measure each card's position relative to grid on mount and resize
Store: cardPositions[] array with {x, y} coordinates
Measure first card's: width, height, borderRadius


Border Position Calculation:

Map FRAME_SEQUENCE to actual pixel coordinates
Creates array of 9 position keyframes matching the sequence
Use fallback positions if measurements not ready


Border Structure (motion.div overlay):

Absolute positioned at top-left initially
Width/height match measured card dimensions
Contains 4 corner bracket divs (not full borders)
Each corner: 3px border with 2 sides (L-shaped brackets)
Border color: blue-500


Corner Bracket Dimensions:

Calculate dynamically: min(cardWidth, cardHeight) * 0.15
Minimum: frameRadius + 8px
Maximum: shortestSide / 1.8
Ensures corners don't overlap


Border Animation:

Animate X: array of 9 x-coordinates
Animate Y: array of 9 y-coordinates
Duration: TOTAL_DURATION (8s)
Times: FRAME_KEYFRAME_TIMES
Ease: SMOOTH_EASE (for synchronized movement)
Infinite loop

Measurement System
Key Implementation Details

Debounced measurement: Use setTimeout with 100ms delay to prevent excessive updates
Initial measurement: Run on component mount via useEffect
Resize listener: Re-measure on window resize
Position calculation:

Get grid's bounding rect
Get each card's bounding rect
Calculate relative positions: cardRect.left - gridRect.left, cardRect.top - gridRect.top


State updates: Only update if positions changed by more than 1px (optimization)
Border radius extraction: Parse from computed styles of first card
Timeout cleanup: Clear measurement timeouts on unmount

Performance Optimizations

Use React.memo for component
Use useMemo for derived calculations (borderPositions, animationKeyframes, cornerLength)
Use useCallback for debounced measurement function and image load handler
Set willChange: "auto" on cards, "transform" on border, "transform, filter" on images
Prevent unnecessary re-renders with precise state comparison
Animation lifecycle management with refs to prevent measurement conflicts

Critical Implementation Notes

The border overlay must be positioned absolutely within the grid container
Border measurements must happen after images load (use onLoad callback with debounced handler)
The border's width, height, and borderRadius must dynamically match the measured card dimensions
The 4 corner brackets create the scanning frame effect (not a full border)
FRAME_KEYFRAME_TIMES and CARD_KEYFRAME_TIMES must be synchronized for proper timing
The FRAME_SEQUENCE determines border position, CARD_SEQUENCE determines card active state
Timing synchronization: 4th card stays active longer (0.75 to 0.88) to match border perception
Use debounced measurements to prevent performance issues during animations
Proper cleanup of timeouts and event listeners to prevent memory leaks
